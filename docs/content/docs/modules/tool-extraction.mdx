title: tool-extraction

`tool-extraction` is the **manager** for function-catalog disclosure
against agentic LLMs. It mirrors `system-prompt-extraction`'s
recon ‚Üí plan ‚Üí execute shape but targets a different surface ‚Äî the
callable-function inventory, parameter schemas, and tool descriptions
exposed by tools-using assistants.

## When to use it

- The target is a function-calling agent (OpenAI Tools, Anthropic
  tool-use, Llama 3.1+ Json Tools, agent frameworks like LangChain
  / Mastra / Vercel AI SDK).
- You want the tool surface, not the system prompt. Many production
  agents leak tool catalogs through pathways that don't disclose
  their system prompt ‚Äî JSON-format requests, schema-shaped probes,
  meta-questions ("what can you help with?"), verbatim recital of
  function definitions.
- You want to enumerate tools before deciding whether to attack any
  individual one. `indirect-prompt-injection` and
  `email-exfiltration-proof` read the resulting catalog to target the
  retrieval and side-effect path.

## When NOT to use it

- The target is a bare chat without tools. There's nothing to
  extract; the manager's probes will all dead-end.
- You want the system prompt, not the tool catalog. Use
  `system-prompt-extraction` instead ‚Äî same shape, different target
  surface.
- You're chasing one specific known tool. Author a focused
  technique; the manager's overhead isn't worth it for a known
  single-target probe.

## Sub-modules it composes

The same recon ‚Üí plan ‚Üí execute pattern as
`system-prompt-extraction`, but with tool-flavoured techniques in the
execute phase:

```yaml
sub_modules:
  - target-profiler                  # T0 recon ‚Äî agentic posture, tool hints
  - name: attack-planner
    see_siblings: true               # planner picks the rotation
  - direct-ask                       # T0 baseline ("what can you do?")
  - instruction-recital              # T0 echo / repeat your tool list
  - format-shift                     # T0 reformat as JSON / schema
  - hallucinated-tool-probing        # T1 enumerate via "did you mean‚Ä¶"
  - fake-function-injection          # T1 forge a schema, observe self-call
  - prefix-commitment                # T1 force a tool-recital prefix
  - delimiter-injection              # T1 mimic the tool-section markers
  - role-impersonation               # T1 fake operator/dev role
  # cognitive techniques ‚Äî applicable, used opportunistically
  - foot-in-door
  - authority-bias
  - narrative-transport
```

The two T1 techniques unique to this manager ‚Äî
[`hallucinated-tool-probing`](/docs/techniques/hallucinated-tool-probing)
and [`fake-function-injection`](/docs/techniques/fake-function-injection)
‚Äî exist specifically because tool-calling models leak through
function-shaped pathways that text-disclosure techniques don't reach.

## Running it

```bash
# Against a hypothetical agent endpoint
uv run mesmer run scenarios/private/your-agent.yaml --verbose

# Against the dvllm research-l1 target (canonical example)
uv run mesmer run scenarios/extract-dvllm-tools.yaml --verbose

# Negative control: dvllm support-l1 has no tools ‚Äî verifies the
# manager doesn't false-positive on a tool-less target.
uv run mesmer run scenarios/extract-dvllm-support-tools.yaml --verbose
```

## Chained recon

`scenarios/extract-system-and-tools.yaml` chains
`system-prompt-extraction` then `tool-extraction` in one run ‚Äî the
executive dispatches both managers in order. Each manager records its
concluded output in Module Conversation History and may update declared
artifacts with curated findings:

```yaml
modules:
  - system-prompt-extraction   # extract hidden prompt first
  - tool-extraction            # then enumerate the tool catalog
```
