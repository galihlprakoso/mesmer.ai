Mesmer is a **cognitive hacking toolkit for LLMs**. It red-teams an LLM
target by running ReAct agents that pick profiling вҶ’ planning вҶ’
execution moves from a tiered attack ladder вҖ” field-level payload
shapes *before* cognitive techniques вҖ” judges every attempt on a 1-10
rubric, and persists a per-target attack graph so every successive run
gets smarter.

## The premise

LLMs are minds, not filesystems. Static prompt banks вҖ” the shape most
red-teaming tools take вҖ” fail against modern targets because a frozen
attack can't adapt to a defense it hasn't seen. Mesmer treats attack
generation as a **search problem over cognitive space**: explore,
exploit, prune, remember.

Every module execution is recorded as an `AttackNode` вҖ” including the
leader's own. The frontier proposer respects the **TAPER tier ladder**
(T0 naive вҶ’ T1 structural вҶ’ T2 cognitive вҶ’ T3 composed), so simple
moves get tried before expensive ones, and wins are attributable to
the cheapest technique that worked.

## What you can do today

<ModuleGrid />

16 shipped modules: 1 leader, 1 profiler, 1 planner, 6 field-level
techniques (T0/T1), 6 cognitive techniques (T2). All YAML, all
declarative, all picked up by `Registry.auto_discover()` on startup.

---

Mesmer is open source, Python 3.10+, uses LiteLLM for all provider
dispatch, and ships as a CLI (`mesmer run`, `mesmer bench`,
`mesmer serve`) plus a FastAPI + Svelte 5 / D3 web UI. Cross-run state
lives outside the repo at `~/.mesmer/` so attack graphs survive across
sessions, and each bench trial writes a self-contained D3 HTML
mind-map you can open offline.
