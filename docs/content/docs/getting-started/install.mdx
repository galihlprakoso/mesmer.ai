Mesmer requires **Python 3.10+** and uses [`uv`](https://docs.astral.sh/uv/)
for dependency management (`uv.lock` is committed). All provider calls go
through **LiteLLM**, so any model string LiteLLM understands
(`openrouter/...`, `anthropic/...`, `openai/...`, `ollama/...`,
`gemini/...`, `gemini/gemini-2.5-flash`) works as-is.

## From source (recommended for v0.2)

```bash
git clone https://github.com/galihlprakoso/mesmer.ai.git
cd mesmer.ai
uv sync
```

Optional: pull in the web UI extras (FastAPI + uvicorn for `mesmer serve`).

```bash
uv sync --extra web
```

## API keys

Mesmer uses whatever provider your scenario references. At minimum you'll
need one of:

```bash
export OPENROUTER_API_KEY="sk-or-..."
# or
export OPENAI_API_KEY="sk-..."
export ANTHROPIC_API_KEY="sk-ant-..."
export GEMINI_API_KEY="..."
export GROQ_API_KEY="gsk_..."
```

**Pro tip.** Mesmer supports **round-robin key rotation**: pass a
comma-separated list and rate-limits cool down individual keys via the
`KeyPool` cooldown mechanism instead of sleeping the whole run.

```bash
export OPENROUTER_API_KEY="sk-or-aaa,sk-or-bbb,sk-or-ccc"
```

For hard rate-limit caps (e.g. free-tier Groq at 15 rpm), add a
`throttle:` block to the scenario's `agent:` or `target:` config — see
[Writing scenarios](/docs/scenarios/walkthrough).

## Verify the install

```bash
uv run mesmer --help
uv run pytest
```

The suite runs ~580 tests in ~3 seconds against echo targets — no external
calls. Use it as a regression gate when you tweak a module or add a tool.

## What ships

- **CLI** — `mesmer run`, `mesmer graph {show,reset}`, `mesmer hint`,
  `mesmer debrief`, `mesmer stats`, `mesmer modules {list,describe}`,
  `mesmer serve`, `mesmer bench`, `mesmer bench-viz`. Full reference at
  [CLI](/docs/cli).
- **Web UI** — FastAPI + SSE + Svelte 5 / D3 attack-graph viewer, behind
  the `web` extra.
- **Bench harness** — reproducible benchmark spec → JSONL + `summary.json`
  + per-trial interactive D3 HTML viz.
