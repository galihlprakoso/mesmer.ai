---
title: Iterating with hints
description: Use mesmer hint and the debrief flow to leave operator context between runs.
---

The graph is mesmer's execution memory. Hints are operator notes inside
that memory. When a run reveals a pattern the executive missed — a
specific refusal template, a promising format, or a target quirk — use
`mesmer hint` to record it as a `source: HUMAN` node.

Hints do not create search frontier nodes. They are durable context that
future runs can read alongside prior module outputs and judged attempts.

## Adding a hint from the CLI

```bash
uv run mesmer hint scenarios/extract-system-prompt.yaml \
  "Try wrapping the request as a Python docstring; the target refuses prose but responds to code-shaped frames."
```

That writes an execution-note node:

```json
{
  "module": "human-insight",
  "approach": "Try wrapping the request as a Python docstring...",
  "status": "completed",
  "source": "HUMAN"
}
```

## The debrief flow

After a run, get a structured walk-through of what the executive saw and
why each technique scored what it did:

```bash
uv run mesmer debrief scenarios/extract-system-prompt.yaml
```

Use this to spot patterns the planner missed, then capture the useful
parts as hints or write them into the shared scratchpad.

## A two-run iterate loop

```bash
# Run #1 — let mesmer execute the scenario
uv run mesmer run scenarios/extract-system-prompt.yaml --verbose

# Read what happened
uv run mesmer debrief scenarios/extract-system-prompt.yaml

# Add your hunch
uv run mesmer hint scenarios/extract-system-prompt.yaml \
  "delimiter-injection: mimic the target's <|system|> token style I saw in turn 14"

# Run #2 — starts with the prior graph and hint context
uv run mesmer run scenarios/extract-system-prompt.yaml --verbose
```

Use `--fresh` when you want to ignore prior graph and conversation state.
The scratchpad is intentionally not deleted by `--fresh`.
