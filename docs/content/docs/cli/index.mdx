The `mesmer` console script wires to `mesmer.interfaces.cli:cli` (Click
group). All commands go through `core/runner.execute_run(RunConfig, ...)`
— the single entry point shared by CLI, web, and bench, so behavior stays
consistent across interfaces.

```
mesmer run               Run one attacker against a scenario.
mesmer graph show        Render the persisted attack graph.
mesmer graph reset       Delete the graph for a scenario's target.
mesmer hint              Pin a human hint as a high-priority frontier node.
mesmer debrief           Interactive post-run Q&A, captures insights.
mesmer stats             Cross-target technique effectiveness.
mesmer modules list      List registered modules.
mesmer modules describe  Show one module's config + prompt.
mesmer serve             Launch the FastAPI + Svelte web UI.
mesmer bench             Run a reproducible benchmark spec.
mesmer bench-viz         Backfill the D3 HTML mind-map for an existing run.
```

## `mesmer run`

Run one attacker agent against a scenario.

```bash
uv run mesmer run <scenario.yaml> [OPTIONS]
```

| Flag | Type | Description |
|---|---|---|
| `--model` | string | Override the attacker model (`openrouter/anthropic/claude-sonnet-4-20250514`, etc.) |
| `--max-turns` | int | Override the scenario's turn budget |
| `--mode` | `trials` \| `continuous` | Override the scenario's `ScenarioMode` |
| `--verbose` / `-v` | flag | Stream every `LogFn` event to stdout with per-event colour + icon |
| `--output` / `-o` | path | Save final report JSON to this file |
| `--module-path` | path (repeatable) | Additional module search paths (layered on top of `BUILTIN_MODULES`) |
| `--hint` | string (repeatable) | Inline human hint — becomes a `NodeSource.HUMAN` frontier node |
| `--hint-file` | path | Load hints from a file (one per line) |
| `--fresh` | flag | Ignore any existing graph under `~/.mesmer/targets/{hash}/` |

**Examples**:

```bash
# Standard run
uv run mesmer run scenarios/extract-system-prompt.yaml --verbose

# Override the attacker model, extend the budget
uv run mesmer run scenarios/extract-system-prompt.yaml \
  --model openrouter/openai/gpt-4o --max-turns 40

# Add hints on the command line
uv run mesmer run scenarios/extract-system-prompt.yaml \
  --hint "try wrapping as a pytest docstring" \
  --hint "target may be fine-tuned on leetspeak"

# CONTINUOUS mode (one long conversation, cross-run persistence)
uv run mesmer run scenarios/extract-system-prompt-continuous.yaml --verbose

# Start fresh — ignore the existing graph
uv run mesmer run scenarios/extract-system-prompt.yaml --fresh
```

## `mesmer graph show`

Print the persisted attack graph for a scenario's target.

```bash
uv run mesmer graph show <scenario.yaml>
```

Renders a summary panel (nodes, best score, run counter) plus three
tables: promising nodes, frontier (next moves), dead ends. For
interactive exploration, use the Web UI (`mesmer serve`).

## `mesmer graph reset`

Delete the persisted graph for a scenario's target. Asks for
confirmation.

```bash
uv run mesmer graph reset <scenario.yaml>
```

Equivalent to `rm ~/.mesmer/targets/{hash}/graph.json`. Use `--fresh`
on `mesmer run` if you want to bypass the graph for one run without
deleting it.

## `mesmer hint`

Add a high-priority human hint for the target. Does not run the loop —
just plants a `NodeSource.HUMAN` node on the frontier.

```bash
uv run mesmer hint <scenario.yaml> "your hint text"
```

Next `mesmer run` explores human hints **before** agent- or judge-
proposed frontier, regardless of TAPER tier.

## `mesmer debrief`

Interactive post-run Q&A. Mesmer reads the graph summary, asks you 3–5
targeted questions, and captures your answers as `NodeSource.HUMAN`
frontier nodes for the next run.

```bash
uv run mesmer debrief <scenario.yaml>
```

Exit with `Ctrl-C`, or type `done` / `quit` / `q` / an empty line.

## `mesmer stats`

Cross-target technique effectiveness — pulled from
`~/.mesmer/global/techniques.json`.

```bash
uv run mesmer stats
```

Shows a table of `technique | attempts | avg_score | best_score` sorted
by average score. Empty if you haven't run any attacks yet.

## `mesmer modules`

Inspect the module registry.

```bash
# List every registered module (name, description, sub-modules)
uv run mesmer modules list [--path <extra-module-dir>]

# Show one module's full config + first 500 chars of system prompt
uv run mesmer modules describe <name> [--path <extra-module-dir>]
```

Modules are auto-discovered from `BUILTIN_MODULES` (sibling of the
Python package at the repo root) plus any paths you pass via `--path`.

## `mesmer serve`

Launch the web UI. Requires the `web` extra (`uv sync --extra web`).

```bash
uv run mesmer serve [OPTIONS]
```

| Flag | Default | Description |
|---|---|---|
| `--host` | `127.0.0.1` | Host to bind to |
| `--port` | `8888` | Port to serve on |
| `--scenario-dir` | `.` | Directory to scan for scenarios |
| `--no-browser` | — | Don't auto-open the browser after startup |

The backend is FastAPI + SSE (streaming `log`, `graph_update`, and
`key_status` events); the frontend is Svelte 5 + D3.

## `mesmer bench`

Run a reproducible benchmark spec: one mesmer module against a pinned
dataset, across a list of target models.

```bash
uv run mesmer bench <spec.yaml> [OPTIONS]
```

| Flag | Type | Description |
|---|---|---|
| `--targets` | `a,b,c` | Only run listed target IDs from the spec |
| `--sample` | int | Limit to first N rows (overrides `budget.sample`; `0` = all) |
| `--trials` | int | Override `budget.trials_per_row` (`1` for smoke) |
| `--output` | path | Override output directory (default `<spec-dir>/../results/`) |
| `--concurrency` | int | Override `budget.concurrency` (max in-flight trials) |
| `--download` | flag | Force re-fetch the dataset even if cached |
| `--no-baseline` | flag | Skip the single-turn baseline arm (mesmer only) |
| `--verbose` / `-v` | flag | Tee per-trial events to stderr |
| `--no-viz` | flag | Skip the auto-generated `{stem}-viz.html` |

Success is decided by the deterministic canary matcher in
`bench/canary.py` — **not** the in-loop LLM judge. Results land in
`benchmarks/results/` as:

- `{stem}-summary.json` — aggregate cells (ASR, TAPER trace)
- `{stem}-README.md` — human-readable markdown table
- `{stem}-{target}__{arm}.jsonl` — per-trial rows with trace envelopes
- `events/{trial_id}.jsonl` — per-trial forensic event log (always written)
- `events/{trial_id}.graph.json` — trial-scoped attack graph snapshot
- `{stem}-viz.html` — interactive D3 per-trial mind-map

See [Bench tracing](/docs/concepts/bench-trace) for the full trace
contract.

## `mesmer bench-viz`

Regenerate the interactive HTML mind-map for an existing benchmark run
(or generate it if `--no-viz` was passed during the run).

```bash
uv run mesmer bench-viz <summary.json> [OPTIONS]
```

| Flag | Description |
|---|---|
| `--offline` | Inline the vendored D3 bundle (~280 KB) so the HTML renders with zero network |
| `--output` / `-o` | Override output path (default `{stem}-viz.html` next to the summary) |

Above `VIZ_INLINE_BYTES_LIMIT` (50 MB of JSON) the generator splits
per-target and writes `{stem}-viz-index.html` as the entry point.

## Environment variables

| Variable | Purpose |
|---|---|
| `OPENROUTER_API_KEY` / `OPENAI_API_KEY` / `ANTHROPIC_API_KEY` / `GEMINI_API_KEY` / `GROQ_API_KEY` / … | Provider auth. Comma-separated for round-robin rotation. |
| Any `${VAR}` referenced in a scenario YAML | Resolved at load time by `core/scenario.py`. Missing values load as empty strings. |

State path (`~/.mesmer/`) is not currently overridable via an env
variable — if you need to relocate it, symlink the directory.
