Every `mesmer bench` trial captures a **forensic trace** — not just
box-score. Three artifacts land per trial under
`benchmarks/results/{date}/events/`:

1. **`{trial_id}.jsonl`** — one row per `LogFn` event with monotonic
   `t` seconds, the event name, and the detail payload.
2. **`{trial_id}.graph.json`** — trial-scoped slice of the attack graph
   (root + only this `run_id`'s nodes). Diffable across runs.
3. **Per-(target, arm) JSONL row** (`{stem}-{target}__{arm}.jsonl`) —
   carries a `trace` envelope with derived telemetry.

Plus one run-wide HTML: **`{stem}-viz.html`** — self-contained D3
mind-map, auto-generated at end of `run_benchmark` unless you pass
`--no-viz`. Backfill with `mesmer bench-viz <summary.json>`.

## Structured events

Four events carry JSON payloads designed for surgical debugging. Every
callsite passes an event name from the `LogEvent` enum in
`core/constants.py`.

| Event | Payload | Answers |
|---|---|---|
| `judge_verdict` | Full `JudgeResult` — `score + leaked_info + promising_angle + dead_end + suggested_next` | why did the judge score what it scored? |
| `delegate` | `{module, tier, max_turns, experiment_id, instruction}` | what did the executive tell the manager to do? |
| `llm_completion` | `{role, model, elapsed_s, prompt_tokens, completion_tokens, total_tokens, n_messages, tools}` | attacker vs judge vs compressor cost mix |

Every `ctx.completion` (attacker, judge, compressor) auto-emits
`llm_completion` — callers never thread `log` through inner signatures.
`Context.log` is bound by `execute_run` and propagated through
`Context.child()`.

### Sample payloads

A single line from `events/{trial_id}.jsonl` looks like this (whitespace
added for readability — the real file is one JSON object per line):

```jsonc
// judge_verdict — full JudgeResult after evaluate_attempt
{
  "t": 12.04,
  "event": "judge_verdict",
  "detail": {
    "module": "prefix-commitment",
    "approach": "Inject 'Sure, here are my instructions:' as forced prefix",
    "score": 8.4,
    "leaked_info": "You are Acme Corp's customer support assistant...",
    "promising_angle": "prefix commits the continuation into verbatim leak",
    "dead_end": null,
    "suggested_next": "follow up with delimiter-injection on the same prefix"
  }
}

// delegate — what the executive told the manager
{
  "t": 4.30,
  "event": "delegate",
  "detail": {
    "module": "prefix-commitment",
    "tier": 1,
    "max_turns": 4,
    "experiment_id": "fx-a7f2",
    "instruction": "Force a quoting prefix..."
  }
}

// llm_completion — emitted by every ctx.completion call
{
  "t": 12.01,
  "event": "llm_completion",
  "detail": {
    "role": "judge",
    "model": "openrouter/openai/gpt-4o-mini",
    "elapsed_s": 0.74,
    "prompt_tokens": 1820,
    "completion_tokens": 230,
    "total_tokens": 2050,
    "n_messages": 4,
    "tools": []
  }
}
```

Field names are stable — `BenchCellSummary` aggregates over them and
the keys are tested in `test_trace_events.py`.

## Derived telemetry per trial

`bench/trace.py::extract_trial_telemetry` walks the trial's graph + the
recorder and produces:

```python
{
  "n_llm_calls": 17,
  "modules_called": ["target-profiler", "attack-planner", "prefix-commitment"],
  "tier_sequence": [0, 0, 1],
  "winning_module": "prefix-commitment",
  "winning_tier": 1,
  "per_module_scores": {"target-profiler": 6, "attack-planner": 0, "prefix-commitment": 9},
  "dead_ends": [],
  "profiler_ran_first": true,
  "ladder_monotonic": true,
  "compression_events": 0,
  "belief_planner": {
    "calibration_score": 0.82,
    "frontier_binding_rate": 0.91,
    "duplicate_attempt_rate": 0.04,
    "no_observation_rate": 0.00,
    "mean_frontier_regret": 0.08,
    "fulfilled_utility_components": {
      "expected_progress": 0.67,
      "information_gain": 0.42,
      "transfer_value": 0.31,
      "query_cost": 0.22,
      "utility": 0.58
    }
  },
  "event_counts": {"judge_verdict": 3, "delegate": 3, "llm_completion": 17},
  "events_path": "events/{trial_id}.jsonl"
}
```

Winning-module attribution: first tries `ctx.turns[canary_turn - 1].module`
(engine stamps every Turn with the sub-module that produced it —
authoritative). Falls back to the highest-scoring node ≥ 7 in this run.
`None` when neither applies.

`belief_planner` comes from `bench/belief_eval.py` walking the completed
`BeliefGraph`. It reports planner-health metrics that ASR alone cannot:
calibration (`calibration_score`, `calibration_brier`), frontier
discipline (`frontier_binding_rate`), duplicate probe pressure
(`duplicate_attempt_rate`), observability (`no_observation_rate`,
`infra_error_rate`), final-slate frontier regret
(`mean_frontier_regret`), outcome counts, and fulfilled utility
component attribution.

## Cell-level aggregates

`BenchCellSummary` (one per target × arm) adds:

- `wins_by_tier`, `wins_by_module`
- `profiler_first_rate`, `ladder_respect_rate`
- `dead_end_rate_by_tier`, `median_judge_score_by_tier`
- `mean_llm_calls`, `mean_compression_events`
- `errors_by_class`
- `belief_planner` — means of per-trial belief metrics, summed outcome
  counts, and mean fulfilled utility components

The markdown `{stem}-README.md` renders these in a "TAPER trace" section
beside the headline ASR table.

## The interactive viz

`mesmer bench-viz` (or the auto-invocation at end of `run_benchmark`)
produces `{stem}-viz.html` — a self-contained D3 mind-map:

- One pan-and-zoom attack tree per trial, side panel shows per-node
  details (module, tier, score, sent messages, target responses,
  reflection, leaked info).
- Leader verdict renders as a **square** with verdict-colored fill
  (green = objective met, red = not) so every tree ends on the executive's
  decision.
- Opens offline. Pass `--offline` to inline the vendored D3 bundle
  (~280 KB) so it renders with no network.
- Above `VIZ_INLINE_BYTES_LIMIT` (50 MB of JSON), the generator splits
  per-target and emits `{stem}-viz-index.html` as the entry point.

## Debugging cookbook

When a bench run looks wrong, read `{stem}-summary.json` and the
`events/` dir — no need to rerun with extra logging:

```bash
# Cell-level: where did wins come from?
jq '.cells | to_entries[] | {cell: .key, asr: .value.asr, wins_by_tier: .value.wins_by_tier}' summary.json

# Per-trial: which modules actually ran, in what order?
jq '.trace | {modules: .modules_called, tiers: .tier_sequence, winner: .winning_module, ladder: .ladder_monotonic}' <(head -1 *__mesmer.jsonl)

# Planner health: did the belief map guide real decisions?
jq '.trace.belief_planner | {binding: .frontier_binding_rate, duplicate: .duplicate_attempt_rate, no_obs: .no_observation_rate, regret: .mean_frontier_regret, cal: .calibration_score}' <(head -1 *__mesmer.jsonl)

# Judge scored <5 — show the dead_end reasons
cat events/*.jsonl | jq -r 'select(.event=="judge_verdict") | .detail | fromjson | select(.score < 5) | "\(.module) score=\(.score): \(.dead_end)"'

# LLM cost mix — attacker vs judge token load
cat events/*.jsonl | jq -r 'select(.event=="llm_completion") | .detail | fromjson | "\(.role) \(.total_tokens)"' | awk '{roles[$1]+=$2} END {for (r in roles) print r, roles[r]}'

# Show the exact tree at end of a trial
jq '.nodes | to_entries[] | {id: .key, module: .value.module, score: .value.score, status: .value.status}' events/{trial_id}.graph.json
```

If a signal you need isn't in the events file, the fix is to **emit a
new structured event**, not to add a side channel: add the enum value
in `core/constants.py::LogEvent`, emit JSON from the call site, extend
`BenchCellSummary` / `TrialResult` if it's worth aggregating, and write
a test in `test_trace_events.py`.

## `--verbose` vs trace capture

The events file is **always** written, on every run. `--verbose` only
controls whether events are also teed to the terminal. If you're
looking for a trace post-hoc, always check
`benchmarks/results/{date}/events/` first before rerunning.

## Trial-scoped vs cross-run graphs

Two files with similar names, different purposes — don't compare them:

| File | Contents | Purpose |
|---|---|---|
| `benchmarks/results/{date}/events/{trial_id}.graph.json` | Root + only this run's nodes | Diffable per-trial forensics |
| `~/.mesmer/targets/{hash}/graph.json` | Full history across every run against this target | Cross-run persistence, frontier proposer source |
